## Understanding Social Engineering
Manupulating Humans to break security. (Hacking people not systems)

### Common Techniques
1. **Urgency** (Your account's data will be deleted in an hour)<br>

   Forces quick decisions<br>
   Prevents victims capability of thinking

2. **Authority** (This is admin from IT. Send your Password ASAP)<br>

   Use Fear or Respect to manupulate

3. **Curiosity** (Salary increment list)<br>

   Exploits human curiosity to manupulate into taking action

4. **Fear** (You violated Rule No. 5.7. click the link below to justify.)<br>

   uses fear to control human decision

## SOC thinking 
If a message trigger strong emotion then it is likely a highly suspicious message.

## Phising
Phishing is a type of social engineering attack where a cybercriminal impersonates a trusted entity like a bank, a well known brand, or a colleague to trick individuals into revealing sensitive information.

### Core Mechanics of Phishing
**The Lure**: Attackers send deceptive messages (emails, texts, or calls) designed to look legitimate.
**The Hook**: The message typically includes a "call to action," such as clicking a link to "verify an account" or opening an attachment like a "unpaid invoice".
**The Catch**: Victims who take the bait may inadvertently download malware or enter credentials into a cloned website that looks identical to a real one.

### Common Types of Phishing
**Email Phishing**: Fake email which pretends to be from Bank/Company/IT team.
**Spear Phising**: It is a targeted phishing attack. Uses real information like- Name, Job description. It is harder to detect.
**Whaling**: Targets bigger individuals like CEO, Executives.
**Vishing**: Phishing conducted via phone calls. Attackers may use *AI voice cloning* to impersonate trusted individuals or official agencies to extract data or demand payments.
**Smishing**:Text messages that often claim issues with a delivery or a bank account to lure users.
**Clone Phishing**: Use replica of a legitimate email previously received by the victim. Swap the original links or attachments with malicious versions.
**Quishing**: Malicious QR codes placed in public or sent via email. Scanning them directs users to fake websites designed to steal credentials.
**Pharming**: A technical attack that redirects a user from a legitimate website to a fraudulent one by hijacking the DNS.

## How Phishing Actually Works (Real Flow)
1. Attacker sends fake email
2. User clicks link or downloads file
3. User enters credentials OR runs malware
4. Attacker gains access <br>

**Example**:
Email: “Your Microsoft account expired. Login now.”

User clicks => fake login page => enters password => attacker steals it

## Indicators of Phishing (SOC Level)
### Email Indicators
- Fake sender (support@micr0soft.com)
- Spelling mistakes
- Urgent language
- Suspicious attachments

### Link Indicators
- Mismatched URL
- Shortened links
- Fake domains
**Example**:
paypal-secure-login.xyz

### Attachment Indicators
- .exe, .js, .docm (macro-enabled)
- Unexpected files

## What a SOC Analyst Checks

### Email Logs
- Sender IP
- Email Header

### User Activity
- Did user click the link
- Did User downloaded the file

### Endpoint Logs
- What file been executed
- Any Powershell activity happened

### Network Logs
- Connectin to suspicious domains

## Example Thinking:
User clicked email => Word file opened => PowerShell executed => possible malware.
