## Logs?
Multiple devices in a network communicate with each other and, most of the time, with the Internet through a router.
These devices continuously generate logs of the activities that occur within them. These devices are sometimes called log sources.<br> 
There are two types of Log devices. they are,
- Host Centric Log sources
- Network Centric Log sources
### Host Centric Log Sources
These log sources capture events that occurred within or related to the host. Devices that generate host-centric logs include Windows, Linux, servers, etc

### Network Centric Log sources
Network-related logs are generated when the hosts communicate with each other or access the internet to visit a website.
Devices that generate network-centric logs are firewalls, IDS/IPS, routers, etc

**Reading Logs and identifying malicious activities has some challenges.**
- Numerous Log Sources
- No centralization
- Limited Context
- Limited Analysis
- Different log format

## To solve those problems we use SIEM
Security Information and Event Management (SIEM) is a security solution
that collects logs from various types of log sources, standardizes their format into a consistent one, correlates them, and detects malicious activities 
and also provides capabilities to enhance security operations using detection rules.

**Features of SIEM**
- Centralized Log Collection
- Normalization of logs
- Correlation of logs
- Real time alerting
- Dashboard and Reporting
- Integration with threat intelligence feeds
- extensive data retention
- powerful searching capabilities

## Log Ingestion
Each SIEM solution has its own way of ingesting the logs. Some common methods used by these SIEM solutions are,
- Agent/ Forwarder
- Syslogs
- Manual Upload
- Port-Forwarding

**To view events in a Windows environment, type Event Viewer in the search bar.** <br>
**To view events in a Linux operated machines, Go inside */var/log/* directory**
