## What is Network Security?
In simple terms, Network security is protecting systems and data from unauthorized access or attacks.

- Servers
- User devices
- Data
- Network traffic

From:

- Malware

## Core Security Components

We focus on 3 MOST IMPORTANT tools:
- Firewall
- IDS (Intrusion Detection System)
- IPS (Intrusion Prevention System)

## Firewall (First Line of Defense)
A firewall is a security device that controls incoming and outgoing network traffic based on predefined rules.
### How it Works:
- Uses rules (allow/deny)
- Checks: IP address, Ports, Protocols

**Example Rules:**
- Allow HTTP (port 80)
- Block unknown IP
### SOC Thinking:
*Firewall decides who can enter the network*
<br>
**What Firewall CAN detect:**
- Unauthorized access attempts
- Suspicious IPs
<br>

**What it CANNOT do well:**
- Detect complex attacks
- Analyze behavior deeply

## IDS (Intrusion Detection System)
*An IDS monitors network traffic and generates alerts when suspicious or malicious activity is detected.* IDS does NOT block, it only trigger alerts.

### Types of IDS:
**Network Intrusion Detection System (NIDS):** NIDS monitors the traffic flow from various areas of the network. The aim is to investigate the traffic on the entire subnet. If a signature is identified, an alert is created.
<br>

**Host-based Intrusion Detection System (HIDS):** HIDS monitors the traffic flow from a single endpoint device. Its aim is to investigate the traffic on that device. If a signature is identified, an alert is created.

### SOC Thinking:
*IDS tells me something is wrong and i must investigate*

**Example Alert:**
*Possible SQL Injection detected*


## IPS(Intrusion Prevention System)

*IPS Detects AND blocks malicious traffic automatically.* IPS is like IDS + action.

### Types of IPS:
**Network Intrusion Prevention System (NIPS):** NIPS monitors the traffic flow from various areas of the network. The aim is to protect the traffic on the entire subnet. If a signature is identified, the connection is terminated.

**Behaviour-based Intrusion Prevention System (Network Behaviour Analysis - NBA):** Behaviour-based systems monitor the traffic flow from various areas of the network. The aim is to protect the traffic on the entire subnet. If an anomaly is identified, the connection is terminated.

**Wireless Intrusion Prevention System (WIPS):** WIPS monitors the traffic flow from a wireless network. Its aim is to protect wireless traffic and stop possible attacks launched from there. If a signature is identified, the connection is terminated.

**Host-based Intrusion Prevention System (HIPS):** HIPS actively protects the traffic flow from a single endpoint device. The aim is to investigate the traffic on a particular device. If a signature is identified, the connection is terminated.

### SOC Thinking:
IPS stops the attack before damage happens.

**Example:**
Detects attack and blocks IP instantly.

## Detection/Prevention Techniques
There are three main detection and prevention techniques used in IDS and IPS solutions. They are,
- **Signature-Based:** This technique relies on rules that identify the specific patterns of the known malicious behaviour. 
This model helps detect known threats. 

- **Behaviour-Based:** This technique identifies new threats with new patterns that pass through signatures. 
The model compares known/normal behaviours with unknown/abnormal behaviours, helping detect previously unknown or new threats.

- **Policy-Based:**	This technique compares detected activities with system configuration and security policies. 
This model helps detect policy violations.
