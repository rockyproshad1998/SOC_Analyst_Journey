## What is Network Security?
In simple terms, Network security is protecting systems and data from unauthorized access or attacks.

### Real Meaning (SOC Perspective)

I am protecting:
- Servers
- User devices
- Data
- Network traffic

From:
- Hackers
- Malware
- Unauthorized users

## Core Security Components

We focus on 3 MOST IMPORTANT tools:
- Firewall
- IDS (Intrusion Detection System)
- IPS (Intrusion Prevention System)

## Firewall (First Line of Defense)
A firewall controls which traffic is allowed or blocked.

### How it Works:
- Uses rules (allow/deny)
- Checks: IP address, Ports, Protocols

**Example Rules:**
- Allow HTTP (port 80)
- Block unknown IP
### SOC Thinking:
*Firewall decides who can enter the network*
<br>
**What Firewall CAN detect:**
- Unauthorized access attempts
- Suspicious IPs
<br>

**What it CANNOT do well:**
- Detect complex attacks
- Analyze behavior deeply

## IDS (Intrusion Detection System)
*IDS Monitors traffic and alerts when something suspicious happens.* IDS does NOT block, it only alerts.

### Types of IDS:
**Signature-Based**
- Detects known attacks
- Like antivirus
<br>

**Anomaly-Based**
- Detects unusual behavior

### SOC Thinking:
*IDS tells me something is wrong and i must investigate*

**Example Alert:**
*Possible SQL Injection detected*


## IPS(Intrusion Prevention System)

*IPS Detects AND blocks malicious traffic automatically.* IPS is like IDS + action.

### SOC Thinking:
IPS stops the attack before damage happens.

**Example:**
Detects attack and blocks IP instantly.

## Real SOC Scenario
### Situation:
**Firewall logs:**
- Blocked IP 203.0.113.5
<br>

**IDS alert:**
- “SQL Injection attempt detected”
<br>

**IPS action:**
- Blocked attacker automatically
### SOC Thinking:
- Firewall blocked unknown IP
- IDS detected attack pattern
- IPS prevented damage
<br>

**Multi-layer defense working together.**
